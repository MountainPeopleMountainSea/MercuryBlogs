对象

##1.1语法
对象可以通过两种形式定义，声明（文字）形式和构造形式。

文字形式

var obj = {
    key : value
}

构造形式

var obj = new Object();
obj.key = value;

一般使用文字形式

##1.2类型
JavaScript中一共有6个主要类型

  • string
  • number
  • boolean
  • null
  • undefined
  • object

注意：这几种基本类型本身并不是对象，而null有时会被当做一种对象类型，用typeof null 会打印出object

因为在JavaScript中二进制前三位都为0就会被判断为object类型，null的二进制都是0，自然前三位也是0，所以在判断时会返回object

###内置对象

  • String
  • NUmber
  • Boolean
  • Object
  • Function
  • Array
  • Date
  • RegExp
  • Error

在JavaScript中，他们实际上只是一些内置函数，这些内置函数可以当作构造函数来使用，可以构造出一个新对象

栗子

var strPrimitive = "I am a string";
console.log(typeof(strPrimitive),strPrimitive);     //string I am a string
console.log(strPrimitive instanceof String);       //false

var strObject = new String("I am a string");
console.log(typeof(strObject),strObject);         //object String {"I am a string"}
console.log(strObject instanceof String);         //true

在使用文字形式来定义时，此时的“I am a string”只是一个字面量，并不是一个对象，并且不可变，如果需要对这个字面量执行一些操作，比如获取长度，就需要将其转化为对象。

幸好，语言会自动将这些字面量转化成一个对应的对象，也就是说，我们可以不需要全用构造方法去创建一个变量，也就是不需要显示创建。

关于typeof与instanceof

typeof一般用于判断某个对象的类型

一般只能返回如下几个结果：”number”、”string”、”boolean”、”object”、”function” 和 “undefined”。

我们也可以用typeof来判断某个变量是否存在

栗子

if(typeof(a) != "undefined"){
    alert('ok');
}

一般不直接用if(a){}来判读，因为如果a不存在，则会报错，但是对于 Array,Null 等特殊对象使用 typeof 一律返回 object，这正是 typeof 的局限性。

在js数组中，若是动态生成的数组，一般不要直接来判断

//判断是不是数组
if(myArr.length != 'undefined') //这个用法有误
if(typeof(myArr.length) != 'undefined')
if(isNaN(myArr.length))   //或者

instanceof用于判断一个变量是否某个对象的实例

栗子

var a = new Array();
alert(a instanceof Array);  //true
alert(a instanceof Object); //true   Array是Object的子类

function fun1(){};
var b = new fun1();
alert(b instanceof fun1);  //true

更重要的一点，instanceof可以用来判断一个实例是否属于它的父类

function test1(){};
function test2(){};
var test = new test1();
console.log(test instanceof test1);  //true
console.log(test instanceof test2);  //false

1.3内容

对象中存储的值一般不会存储在对象内部，存储在对象内部的是这些属性的名称。它们就像指针一样，指向这些值真正的存储位置。

一段代码

var obj = {
    a:2
};
obj.a;     //2
obj["a"];  //2

要访问obj中a的值，有两种方式，使用.操作符或[]操作符。.操作符通常称作属性访问，[]操作符称作键访问。他们实际上访问的是同一个位置，我们通常使用的是属性访问。

两种方式的区别：

.操作符需要满足标识符的命名规范，而在[]中可以接受任何UTF-8/Unicode字符串作为属性名。比如说，属性名如果是hello-world，那就只能用[]操作符来访问，因为hello-world不是一个规范的标识符属性名。

###可计算属性名

ES6新增了可计算属性名

栗子

var prefix = 'feel';
var obj = {
    [prefix+'good']:'123',
    [prefix+'bad']:'456
}
console.log(obj['feelgood']);  //123
console.log(obj['feelbad']);   //456

可计算属性名最常用的场景是ES6的符号Symbol

数组

数组也支持[]操作符访问，传入的是数值的下标，也就是该数值的存储位置（索引位置）

当然，数组也是对象，虽然每个下标都是整数，但仍然可以给数组添加属性

var arr = ["aaa","123"];
arr.newOne = "bbb";
console.log(arr.length); //2
console.log(arr)   //0: "aaa" 1: "123" newOne: "bbb"

可以看到，虽然给数组添加了属性，数组的长度length并没有发生改变。
