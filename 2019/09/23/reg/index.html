1.什么是正则表达式

正则表达式（Regular Expression），计算机科学的一个概念。正则表达式使用单个字符串来描述、匹配符合某个句法规则的字符串。在很多文本编译器中，正则表达式常用来检索、替换匹配的文字。

2.正则表达式语法

2.1修饰符

  • g：global 执行一个全局匹配
  • i：ignore case 忽略大小写
  • m：multiple line 多行匹配

2.2元字符

元字符 (* + ? $ ^ . | \ ( ) { } [ ])

2.3字符类[]

手机号正则的栗子：/^1[3456789]\d{9}$/，[3456789]表示3456789中任意一个数字都可。

2.4字符类取反[^]

还是手机号正则的栗子：/^1[^012]\d{9}$/，[^012]表示除了这三个其它数字都可以。

2.5范围类[-]

设置范围如：[a-z]、[0-9]，可以连写[a-z0-9]

2.6预定义类

目的：方便书写

字符 等价类 含义
. [^\n\r] 匹配除了回车和换行符外的所有字符
\d [0-9] 匹配数字字符
\D [^0-9] 匹配除了数字字符
\w [a-zA-Z_0-9] 匹配单词字符（数字、单词、下划线）
\W [^a-zA-Z_0-9] 匹配非单词字符
\s [\t\n\x08\f\r] 匹配空白符
\S [^\t\n\x08\f\r] 匹配非空白符

有了这些预定义类，会更加方便我们书写正则表达式。

2.7边界

字符 含义
^ 匹配开始字符
$ 匹配结束字符
\b 单词边界，栗子：/\bis\b/匹配is单词
\B 非单词边界
* 匹配前一位字符开始0次或多次，例如12*能匹配1和123，等价于{0,}
+ 匹配前一位字符开始1次或多次，例如12+能匹配12和123，但是不能匹配1，等价{1,}

2.8量词

字符 含义
? 出现0次或1次
* 出现0次或任意次
+ 出现1次或任意次（至少1次）
{n} 出现0次或n次
{n,m} 至少n次，最多m次
{n,} 至少出现n次（+的升级版）
{0,n} 最多出现n次

如果没有量词，若要匹配4位数字，这样写/\d\d\d\d/，如果要匹配100位，这样写就不行了，有了量词，就可以简写成/\d{4}/
