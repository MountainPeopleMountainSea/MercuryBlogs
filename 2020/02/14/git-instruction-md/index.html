git常用指令

Feb 14, 2020 | git

git作为现在常用的代码版本管理工具，掌握一些常用的git指令时必不可少的，下面记录了一些常用的git指令。

1.创建版本库

  • 项目初始化：git init

  • 当前仓库状态：git status

  • 提交修改的文件:git add filename（添加指定文件） || git add -A（添加当前所有修改的文件）

  • 提交更改：git commit -m “msg”

2.版本管理

  • 查看commit提交历史：git log || git log –pretty=onelien（只输出版本id和commit信息）

  • 回滚到上一个版本：git reset –hard HEAD^（上一个是^，上上个就是^^）

  • 回滚到指定版本：git reset –hard id（只需要前几位即可，可以回到“未来的版本”）

  • 找不到版本号的回滚办法：git reflog（记录了每一次命令，信息前面就是版本号，简直就是实实在在的后悔药）

  • 工作区与暂存区：当前目录就是我们的工作区，工作区中有一个隐藏目录.git,叫做git的版本库，里面就存了暂存区（stage），git add将修改的内容添加到暂存区中，git commit再将暂存区里的内容提交到当前的分支。

  • 把暂存区的修改退回到工作区：git reset HEAD filename

3.远程仓库

  • 关联一个远程库：git remote add origin git@github.com:qshjd/gittt.git

  • 第一次推送master分支的所有内容：git push -u origin master

  • 本地修改提交后推送给远程库：git push origin master

4.分支管理

  • 查看分支：git branch

  • 创建分支：git branch xxx

  • 删除分支：git branch -d xxx

  • 切换分支：git checkout xxx || git switch xxx（新版本提供）

  • 创建并切换分支：git checkout -b xxx || git switch -c xxx

  • 合并分支：git merge dev（假设当前在master分支上）
